Videos Details

GET

https://kailywebapi-default-rtdb.firebaseio.com/videos/<--eTag-->/ <--parameter-->/ <--sub-parameter-->.json

Description:-

"eTag" can be Found Here
"➣" represents Parameter &
"•" represents Sub-Parameter


◈ eTag
  • eTag is uninque number given to each video. [always between 0-49] where 0 will always be the latest video and 49 will the last video in latest 50 videos.
    Note: If you want Change order and make 0 as last video and 49 as latest video then you may proceed here

    • ◈ Parameter :-
        ➣ Snippet
    1. • channelId : returns Channel Id
    2. • channelTitle : returns Channel Title
    3. • channelUrl : returns Channel Url
    4. • categoryId : returns category Id
    5. • title : returns Video Title
    6. • description : returns video description
    7. • defaultAudioLanguage : returns video Audio Language
    8. • tags : returns video tags
    9. • thumbnails : returns video thumbnails
    10. • liveBroadcastContent : returns video Broadcast type
    11. • publishedAt : returns video published time
        12. ➣ Statistics
    12. • likeCount : returns Video Like Count
    13. • dislikeCount : returns video Dislike Count
    14. • viewCount : returns video View Count
    15. • commentCount : returns video Comment Count
    16. • favoriteCount : returns video favoriteCount


    Videos Details Order = Oldest

    GET

    https://kailywebapi-default-rtdb.firebaseio.com/order=oldest/videos/ <--eTag-->/ <--parameter-->/ <--sub-parameter-->.json

    Description:-

    From this Endpoint you will get the same result as Video Details.
    The difference is that the order with will opposite. the Latest video can be fund with eTag 49 and the oldest video will be have eTag 0
    Note: All Parameter and sub-Parameter will remain Same
